LLM(Claude 같은 AI 코드 에이전트)이 흔히 저지르는 실수를 줄이기 위한 행동 가이드라인이다.
프로젝트별 지침과 함께 병합해서 사용하면 된다.

 트레이드오프:
이 규칙들은 속도보다 신중함을 우선한다.
간단한 작업에서는 상황에 맞게 유연하게 판단하라.

1. 코딩 전에 생각하기 (Think Before Coding)

코드를 바로 작성하지 말고 먼저 문제를 명확히 이해하라.

 - 가정을 숨기지 말고 명확히 드러내라
 - 불확실하면 질문하라
 - 가능한 접근 방법을 여러 개 제시하라
 - 더 단순한 해결책이 있다면 그걸 제안하라


2. 단순함 우선 (Simplicity First)

불필요하게 복잡한 코드를 만들지 마라.

 - 필요한 것만 작성하라
 - 미래를 위한 과한 확장성/유연성 추가 금지
 - 불필요한 추상화 금지
 - 짧고 명확한 코드를 선호

3. 외과적 수정 (Surgical Changes)

필요한 부분만 정확히 수정하라.

 - 요청된 부분만 변경하라
 - 관련 없는 코드 개선 금지
 - 기존 코드 스타일 유지
 - 변경으로 인해 깨진 부분만 정리

4. 목표 기반 실행 (Goal-Driven Execution)

작업을 지시할 때는 무엇을 할지보다
 - 성공 기준을 정의하라

예시:

 나쁜 예

"버그 수정해"

 좋은 예

"실패하는 테스트를 만들고, 그 테스트를 통과하게 수정해"

 나쁜 예

"검증 로직 추가해"

 좋은 예

"잘못된 입력에 대한 테스트를 작성하고, 통과하게 만들어라"
